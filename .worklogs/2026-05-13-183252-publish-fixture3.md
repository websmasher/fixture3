# Summary

Created the GitHub repository, pushed `main`, published the crates.io install stub `fixture3 v0.1.0`, and pushed tag `v0.1.0` to trigger binary release artifacts.

Fixed CI after discovering `guardrail3-rs` is not published to crates.io and the Guardrail3 repository is not readable from this repo's default GitHub Actions token.

# Decisions Made

- Created `https://github.com/websmasher/fixture3` as a public repository because the crates.io package metadata points to that public release URL.
- Published only the `fixture3` install stub package to crates.io.
- Pushed tag `v0.1.0` so the binary release workflow can create the artifacts used by `cargo binstall fixture3`.
- Kept local `scripts/verify-all.sh` as the full gate with G3RS. GitHub CI runs all local checks except the private G3RS tool.
- Left the implementation package `fixture3-cli` unpublished.
